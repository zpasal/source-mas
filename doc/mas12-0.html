12. INSTALLING AND PORTUNG TO ANOTHER OS
Mas binaries and source can be downloaded from 

www.sf.net/projects/mas-pic16c84

For now, only binaries for Win32 and LINUX/unix are distributed.

Win32 installing:

Easiest way is to download install-mas-win32.exe. Just follow on screen instructions.

Another way is to download source file to build:

  • You need c++ compiler that is (mutch as can be) written with standard. I preffer MinGW and source for mas (source-mas.zip)
  • Unzip surce to some directory
  • run 'make CFG=Relese' from command line to build mas.exe. If you wannt mas.exe with debug information run 'make CFG=Debug'..NOTES: makefile is written for g++ compiler. If you have another compiler you must CHANGE makefile!
  • if you need to put include/device files in some another directory edit mas.ini. First line is path to include directory, and second line is path to device directory. Mas.ini MUST be in same directory where is mas.exe.


LINUX/UNIX installing:
 

  • You need source for mas (source-mas.tar.gz)
  • Unzip/untar surce to some directory
  • run 'make CFG=Release' from command line to build mas executable. If you wannt executable with debug information run 'make CFG=Debug'.
  • if you need to put include/device files in some another directory edit mas.ini. First line is path to include directory, and second line is path to device directory. Mas.ini MUST be in same directory where is mas executable.


Mas can be easily ported to another platforms that support standard C++ compiler. It can be ported without any changes if following conditions are met:

  sizeof(int) == 32
  sizeof(short) == 16
  sizeof(char) == 8

Most reasons for this is module for creating output formats ('object.cpp'). Binaries for Win was compiled with gcc 3.1.
